Task 1B - Design Documentation

1. Visual & Interface Design

The design shouldn't contain information which is irrelevant or rarely needed. Every piece of extra information competes with the relevant information and diminishes their relative visibility. So, the use of images which have little or no relevance to the product or brand should be avoided.

The proposed solution is a responsive website that allows customers to browse locally produced goods, learn about producers, and place orders digitally. Producers are able to update product listings and manage stock levels through a simple interface. The design prioritises usability and clarity so that both customers and producers can complete their tasks quickly and efficiently.

The interface uses a structured layout with clearly separated sections such as navigation, product listings, and information panels. White space is used between elements to improve readability and reduce visual clutter. This allows users to easily scan product listings and locate important information such as prices, availability, and ordering options.

A visual hierarchy is created through the use of headings, font sizes, and content positioning. Important information such as product names, prices, and purchase buttons are displayed more prominently, while supporting details such as product descriptions are placed beneath them. This guides users naturally through the content and reduces the amount of effort required to find key information.

Consistency is maintained across the system through the use of the same colour palette, typography, navigation bar, and button styles on every page. This ensures users can quickly understand how the system works and reduces confusion when moving between different parts of the website.

The design follows a minimalist approach by only displaying information that is relevant to the user's current task. For example, product pages focus on product details and ordering options, while producer pages focus on information about local suppliers. This prevents unnecessary distractions and helps users focus on completing their tasks efficiently.

The system provides clear feedback messages to inform users about system actions. For example, confirmation messages are displayed when items are added to the cart, orders are placed, or stock levels are updated. Error messages are also provided if invalid input is entered, ensuring users understand what has gone wrong and how to correct it.

Relevant images are used to support product listings and producer profiles. Product images allow customers to quickly identify items before purchasing, while images of producers help promote the local community and strengthen the connection between customers and local suppliers.

The interface has been designed with accessibility in mind. Clear fonts, sufficient colour contrast, and simple navigation ensure the system can be used by a wide range of users. Buttons are clearly labelled and important actions are easy to locate, reducing cognitive load for users.

2. Data Requirements

The system will store and process several variables including product names, prices, stock quantities, user details, and order information. These variables are used to manage product listings, process customer orders, and update stock levels when purchases are made.

Appropriate data types are used to ensure data is stored correctly and efficiently. For example, product names and user details are stored as text strings, prices are stored as decimal values to support currency, and stock quantities are stored as integers. Dates are used to record when orders are placed.

The system will use a structured database consisting of several related tables. These include a Users table to store account information, a Products table to store product listings and stock levels, a Producers table to store details about suppliers, and an Orders table to store purchase records. Relationships between these tables allow the system to link customers, products, and orders.

3. Algorithms

The order process begins when a user proceeds to checkout. The system checks that items exist in the cart and verifies stock availability for each product. If sufficient stock exists, the order total is calculated and the order is stored in the database. Product stock levels are then updated and the user receives a confirmation message. 

START

INPUT userID
GET items from cart

IF cart is empty
    DISPLAY "Cart is empty"
    STOP
ENDIF

FOR each item in cart
    CHECK stock availability

    IF stock is insufficient
        DISPLAY "Item out of stock"
        STOP
    ENDIF
ENDFOR

CALCULATE order total

STORE order in database
UPDATE product stock levels
CLEAR user cart

DISPLAY "Order successfully placed"

STOP

Producers can update the stock levels of their products through the management interface. After entering a new stock value, the system validates the input and updates the product record in the database. The updated stock level is then immediately reflected on the product listing visible to customers. 

START

INPUT producerID
INPUT productID
INPUT newStockLevel

IF producer is authenticated
    FIND product in database
    UPDATE stock value
    SAVE changes
    DISPLAY "Stock updated successfully"
ELSE
    DISPLAY "Access denied"
ENDIF

STOP

When a user selects a product and clicks the "Add to Cart" button, the system checks that the product exists and that sufficient stock is available. The selected quantity is then added to the user's cart and the cart total is updated. A confirmation message informs the user that the item has been successfully added. 

START

INPUT productID
INPUT quantity

IF product exists in database
    IF stock >= quantity
        ADD product to user's cart
        UPDATE cart total
        DISPLAY "Product added to cart"
    ELSE
        DISPLAY "Not enough stock available"
    ENDIF
ELSE
    DISPLAY "Product not found"
ENDIF

STOP

5. User Journey

A typical user journey begins when a customer visits the website and browses the product catalogue. The user selects a product to view its details and chooses a quantity to add to their cart. The user then reviews the cart and proceeds to checkout where the order is confirmed. Once the order is placed, the system stores the transaction and updates product stock levels. 

Example User Journey - Customer Purchasing a Product

1. User opens the website.
2. User browses available local products on the product catalogue page.
3. User selects a product to view more details.

5. User opens the shopping cart page to review selected items.
6. User proceeds to checkout.
7. User confirms the order and payment details.

7. Test Strategy

The system will be tested using a combination of unit testing, integration testing, and system testing. Individual components such as the product search, cart calculation, and stock update functions will first be tested separately. These components will then be tested together to ensure correct communication between the front-end interface and database. Finally, the entire system will be tested from a user perspective to ensure that ordering and stock management functions operate correctly.
