7. Wider Issues, Risks & Mitigation

For each risk: identify the risk → explain the impact → explain the mitigation.
Focus on realistic, technical mitigations — not vague statements like “use security”.

Risk: unauthorised account access
Impact: data breach, reputational damage, financial loss
Mitigation: hashed passwords, HTTPS, secure authentication, rate limiting


Risk: personal data misuse / GDPR breach
Impact: legal penalties, loss of trust
Mitigation: data minimisation, user consent, secure database storage


Risk: incorrect calculations / incorrect system output
Impact: unsafe decisions, financial loss, user harm
Mitigation: validated inputs, tested formulas, verification checks


Risk: system not usable by disabled users
Impact: exclusion, legal non-compliance (Equality Act)
Mitigation: WCAG guidelines, high contrast UI, keyboard navigation, screen reader compatibility


GDPR • Computer Misuse Act • Equality Act • industry-specific regulation
Explain how the system complies.

Risk: users unable to resolve issues
Impact: reduced satisfaction, abandonment
Mitigation: help documentation, support contact, maintenance plan

9. KPIs

KPIs must be specific, measurable, achievable, relevant and time-bound (SMART).

registration success rate (e.g. 95%+)
task completion efficiency (e.g. under 3 steps)
system uptime (e.g. 99%+)
page load speed (e.g. under 3 seconds)


calculation accuracy (e.g. 98%+)
validated input success rate


user testing feedback (e.g. 80%+ positive response)
usability testing outcomes


each KPI should directly relate to a functional or non-functional requirement

weak kpi: system should be fast
strong kpi: system must load within 3 seconds under normal operating conditions
