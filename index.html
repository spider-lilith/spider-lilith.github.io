Task 1B - Design Documentation

1. Visual & Interface Design

The design shouldn't contain information which is irrelevant or rarely needed. Every piece of extra information competes with the relevant information and diminishes their relative visibility. So, the use of images which have little or no relevance to the product or brand should be avoided.

Minimalism - no unnecessary content, like irrelevant images or clutter
Memory Load Reduction - visible navigation, clear buttons, labels, menus, breadcrumbs
Feedback Visibility - confirmation messages, error messages, loading indicators

Describe Layout - clear structure, logical grouping of elements, visual hierarchy
White Space - line spacing (leading) important for readability. too small - hard to read and follow along, too big - wander away from the text

What do users see first? - use of size, colour, contrast to indicate importance and guide users
Primary v Secondary actions - primary actions (e.g. submit, buy) should be more visually prominent than secondary actions (e.g. cancel, learn more)

Familiar Patterns - use common UI patterns (e.g. hamburger menu, search bar) to leverage users' existing knowledge and reduce learning curve
Consistency - consistent use of colours, fonts, button styles across the design to create a cohesive experience
Why does it improve usability? - reduces cognitive load, helps users form accurate mental models, and makes the interface more predictable Common Conventions - follow established conventions, like logo in top left, navigation at top or left, clickable elements look clickable, F-Shaped scanning pattern for text-heavy pages

Colour Contrast - ensure sufficient contrast between text and background for readability
Keyboard Navigation - design should be fully navigable using a keyboard for users with mobility impairments
Screen Reader Compatibility - use semantic HTML and ARIA labels to ensure content is accessible to screen readers

justify colour choices, explain why clutter is avoided, imagery is relevant and purposeful

mention help features such as tooltips, FAQs, contact support, user guides, and explain how they enhance usability and decrease user frustration


1. VISUAL / INTERFACE DESIGN 1.1 DESIGN OVERVIEW & OBJECTIVES
Explain:
• Minimalism • Memory load reduction • Feedback visibility • Accessibility The interface is designed to be:
• Clear and intuitive • Accessible • Consistent • Efficient The design minimises user memory load by ensuring that:
• Key actions are always visible • Important information does not require users to navigate between multiple pages • Instructions and feedback are displayed contextually The system provides continuous feedback so users are always aware of:
• What action has been taken • What is currently processing • What the outcome of their action is Only relevant information is shown on each screen to avoid visual overload and reduce cognitive strain.

1.2 LAYOUT & VISUAL STRUCTURE
Discuss:
• Grid structure • Leading (line spacing) • Padding • Section separation • Primary vs secondary actions Make it analytical.
The interface uses a structured layout such as:
• Grid-based design • Clearly separated content sections • Consistent alignment of elements White space is used effectively to:
• Separate functional areas (navigation, content, actions) • Improve readability • Prevent overcrowding Leading (line spacing) is balanced to ensure text is readable without creating excessive empty space. Primary actions are visually emphasised using:
• Size • Position • Colour contrast Secondary actions are visually reduced to prevent distraction.

1.3 NAVIGATION & USER FLOW
structure like:
• Key screens • How users move between them • Why navigation is predictable Navigation follows common conventions to reduce learning time, such as:
• Persistent top navigation bar or sidebar • Clearly labelled buttons • Logical page hierarchy Example user flow:
User lands on Home → Selects key function → Completes form/input → System validates input → Feedback displayed → Redirected to confirmation or results page The design ensures predictable navigation patterns so users do not need to remember previous steps. 1.4 CONSISTENCY & BRANDING
this separate: • Fonts • Colours • Button styles • Iconography • Industry conventions Consistency is maintained through:
• Consistent font family and scaling • Standardised button styles • Repeated iconography • Uniform spacing rules Industry conventions are followed where appropriate to ensure familiarity.

1.5 FEEDBACK & ERROR PREVENTION
Keep this separate: • Real-time validation • Error messages • Loading states • Confirmation messages The system provides: • Real-time form validation • Clear error messages explaining the issue • Confirmation messages for successful actions • Loading indicators during processing Error messages are specific and constructive, guiding users toward resolution rather than simply stating failure.

1.6 ACCESSIBILITY final subsection:
• WCAG principles • Contrast ratios • Semantic structure • Responsive design The design supports accessibility by:
• Providing sufficient colour contrast • Allowing text scaling • Using semantic structure for screen readers • Ensuring keyboard navigation compatibility • Designing responsive layouts for multiple devices
